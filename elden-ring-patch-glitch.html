Elden Ring Patch 1.15 Glitch: Modded Co-Op Saves Corrupting After Update

By Gaming Trends Desk • July 13, 2026 • 3 min read

A critical issue has surfaced following FromSoftware's release of the **v1.15 patch** for Elden Ring. Players utilizing custom local co-op mods are reporting immediate, permanent **save file corruptions** upon booting the game post-update.

The Source of the Glitch

The version 1.15 patch introduced several backend changes to save verification protocols, designed to prepare the game for upcoming server security updates. However, these checks conflict with custom save-redirecting scripts used by the community's popular multiplayer extensions. When the game detects modified file structures, it flags the save file as corrupted and refuses to load, locking out players with hundreds of hours of progress.

"My 300-hour save file is completely gone. The game won't even boot up in offline mode anymore without throwing a verification error."
— Community forum report

Workarounds and Quick Fixes

Mod authors have quickly responded, advising players to disable all automated script injections immediately and avoid launching the game until a mod compatibility update is deployed. For players who have already encountered the error, restoring a local backup copy of their `.sl2` save file created prior to installing the patch seems to be the only viable recovery path.

Official Response

While FromSoftware has not commented directly on third-party mod issues, they released a statement reminding players that modifying local files is not officially supported and can lead to online play restrictions. They recommend reinstalling a clean, unmodded client to resolve stability bugs.
